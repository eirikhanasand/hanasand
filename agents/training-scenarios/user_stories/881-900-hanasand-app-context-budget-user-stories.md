# User Stories 881-900: Hanasand App Context Budget And Fast Resolution Checks

These stories test whether Hanasand can keep making useful, reviewable progress without spending context like an opaque terminal session. The user prompts are intentionally vague, impatient, or incomplete. The app should send lean context, produce concise visible output, expose pending files, and avoid fake systems.

## 881. "The Page Is Too Much"

Perspective: a founder overwhelmed by a bloated draft.

User ask: "this is too much, make the page obvious"

Success signals:
- The response reaches review quickly.
- The next step is plain.
- The request uses lean context.

Failure signals:
- More explanation than output.
- No pending files.
- Large context payload for a small change.
