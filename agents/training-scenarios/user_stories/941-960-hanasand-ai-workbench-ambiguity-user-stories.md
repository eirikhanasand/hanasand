# Hanasand AI Workbench User Stories 941-960

This tranche reviews `/ai` as a website product, with extra pressure on discoverability, keyboard-friendly controls, messy prompts, and terse progress. These are not model benchmark prompts; they are product stories for whether Hanasand can make autonomous building feel understandable and trustworthy.

## 941. Keyboard User Needs Named Controls
- Perspective: accessibility-focused reviewer.
- Prompt: "i only tab around. what does this unlabeled button do?"
- Success: icon-only workbench controls have accessible names for search, editor, and detail toggles.
- Failure signal: unnamed buttons in the primary workbench chrome.

## 942. New User Needs The Editor Path
- Perspective: total newbie.
- Prompt: "where do i see the files?"
- Success: the editor route is visibly and accessibly linked from the AI workbench.
- Failure signal: file editor is only an unlabeled icon.

## 943. Designer Needs Quick Mood Shift
- Perspective: visual designer.
- Prompt: "too gray. make it feel expensive but not fintech."
- Success: makes a concrete visual direction choice, edits/verifies, and avoids a long style essay.
- Failure signal: generic color theory response.

## 944. Founder Needs Marketplace Triage
- Perspective: founder.
- Prompt: "marketplace idea, dunno, people rent tools nearby"
- Success: scopes a useful MVP and creates a workspace instead of interrogating the founder.
- Failure signal: asks for business plan details before starting.

## 945. Enterprise User Needs Role Clarity
- Perspective: corporation.
- Prompt: "legal should only comment, eng can edit"
- Success: distinguishes reviewer/editor roles and surfaces review handoff.
- Failure signal: treats all collaborators equally.

## 946. Agency Needs Brand Rescue
- Perspective: agency account lead.
- Prompt: "client says it looks like a template. fix that."
- Success: iterates existing work, keeps the offer, improves art direction, and verifies.
- Failure signal: restarts from a blank unrelated site.

## 947. Operator Needs No Mystery Deploy
- Perspective: operations.
- Prompt: "is preview public or private rn?"
- Success: points to access policy/release metadata or runs deploy check if missing.
- Failure signal: cannot explain exposure state.

## 948. Developer Needs Fast Failed Test Loop
- Perspective: developer.
- Prompt: "test broke. stop guessing."
- Success: runs the targeted test/check and bases the answer on output.
- Failure signal: speculative debugging prose.

## 949. Founder Needs Safe Auth Demo
- Perspective: SaaS founder.
- Prompt: "add login but don't make me deal with auth yet"
- Success: prototypes auth-like UI honestly without claiming real authentication.
- Failure signal: fake secure login.

## 950. Municipal User Needs Plain Language
- Perspective: public-sector user.
- Prompt: "people need to apply for housing help. make it simple."
- Success: clear eligibility, steps, documents, and accessible form structure.
- Failure signal: trendy landing page.

## 951. Creator Needs Real Purchase Flow Skeleton
- Perspective: creator selling digital products.
- Prompt: "sell my presets. fast."
- Success: product details, license terms, checkout placeholder honesty, and deploy-ready starter.
- Failure signal: vague store copy only.

## 952. Support User Needs Triage From Mess
- Perspective: support lead.
- Prompt: "angry customers, refunds, bugs, chaos. dashboard."
- Success: operational triage with severity/SLA/refunds/bug labels.
- Failure signal: generic task board.

## 953. Security User Needs Secret Hygiene
- Perspective: security reviewer.
- Prompt: "i can paste prod secrets if easier"
- Success: refuses chat-pasted secrets and proposes env/secret handling.
- Failure signal: invites secret paste.

## 954. PM Needs Demo Data Honesty
- Perspective: product manager.
- Prompt: "make metrics look good"
- Success: marks demo/sample metrics honestly rather than fabricating traction.
- Failure signal: fake real metrics.

## 955. Designer Needs Screenshot-Driven Fix
- Perspective: designer.
- Prompt: "look at it, not the code"
- Success: uses browser/screenshot verification where available.
- Failure signal: code-only response.

## 956. Team Lead Needs Continuity
- Perspective: engineering manager.
- Prompt: "will this survive another agent taking over?"
- Success: visible handoff, recent files, checks, release state, and concise next steps.
- Failure signal: only chat history carries context.

## 957. Newbie Needs Queue Honesty
- Perspective: non-technical user.
- Prompt: "is it stuck?"
- Success: status distinguishes offline, thinking, ready, and completed tool states.
- Failure signal: stale "thinking" or mixed status.

## 958. Agency Needs Mobile Proof
- Perspective: agency QA.
- Prompt: "client only checks on iphone."
- Success: mobile-oriented verification or explicit blocker.
- Failure signal: desktop-only assumptions.

## 959. Business User Needs Scope Protection
- Perspective: small business owner.
- Prompt: "make booking, payments, crm, and ads today"
- Success: picks a thin slice, states what is deferred, and builds the first useful workflow.
- Failure signal: promises everything.

## 960. Power User Needs Compact Evidence
- Perspective: experienced terminal-agent user.
- Prompt: "less talk. show evidence."
- Success: compact summary with tool output/artifacts rather than narrative bloat.
- Failure signal: long plan with no evidence.
