# Scraper Roadmap This is a long-running production build, not a one-shot scaffold. The main objective is a modular CTI collection service that can answer requests like "find current intelligence on APT29" across normal web, public channels, and approved darknet metadata sources. ## North Star Given an intelligence request, the scraper should: - understand the entity and aliases; - select relevant sources across adapters; - schedule safe collection with per-source policy; - preserve raw evidence and provenance; - extract entities, IOCs, TTP candidates, victims, and events; - deduplicate and correlate across sources; - expose API-ready results to the CTI application; - show confidence, freshness, and "why we believe this." ## Enterprise Baseline This system must be designed like a global CTI platform component: - multi-tenant by contract, even if the first deployment is single tenant; - API-versioned from the start; - auditable collection decisions and operator actions; - policy-driven source governance; - pluggable storage, queue, search, and graph backends; - bounded resource usage per tenant, request, source, adapter, and worker pool; - immutable evidence and reproducible extraction; - separation between raw evidence, normalized captures, extracted claims, and analyst-reviewed intelligence; - operational SLOs for freshness, queue age, adapter error rate, source health, and extraction confidence; - security posture suitable for hostile content handling. ## Phase 0: Control Plane - Coordination files and agent ownership. - Bun/TypeScript service scaffold. - Typed source, task, capture, request, and pipeline contracts. - Health/API foundation. - Safety policy. - Tests and type-checks. ## Phase 1: Real Clear-Web Collection - Production source registry with Postgres schema. - RSS adapter with robust feed parsing, ETag/Last-Modified, pagination, and source health. - Static web adapter with canonicalization, robots/legal notes, HTML extraction, link discovery, and change detection. - Frontier scheduler with per-source concurrency, backoff, retry budgets, queue aging, and freshness scoring. - Raw immutable evidence store. - Tenant-aware collection budgets and API request IDs. ## Phase 2: Intelligence Request Workflows - `/intel/plan`, `/intel/run`, `/intel/status/:id`, `/intel/results/:id`. - Actor alias expansion for APT groups, ransomware groups, malware, CVEs, and victims. - Request-scoped collection budgets. - Analyst approval flow for high-risk sources. - Query-to-source routing: RSS, static web, APIs, public Telegram, darknet metadata. ## Phase 3: Extraction And Correlation - IOC extraction with validation and normalization. - Entity extraction and resolution for actors, aliases, malware, victims, sectors, countries, infrastructure, and CVEs. - Incident candidates with evidence clusters. - Confidence scoring and review reasons. - ATT&CK technique candidate mapping with provenance. - STIX 2.1 export contracts. ## Phase 4: Public Channel And Dynamic Collection - Public Telegram adapter using official/legal APIs only. - Playwright dynamic-page adapter with isolated worker pool. - Screenshot capture where appropriate. - Strict memory and browser concurrency controls. ## Phase 5: Darknet Metadata Collection - Tor/I2P/Freenet metadata-only adapters. - Approved proxy boundary. - No stolen-file downloads. - Actor/victim/date/sector/country/data-type metadata extraction. - URL/content/screenshot hashes, not leaked contents. - Manual approval gates for new sources. ## Phase 6: Scale And Operations - Production queue backend. - Postgres persistence with migrations. - Optional OpenSearch/vector/graph integration through clear interfaces. - Metrics, structured logs, alerts, and dashboards. - Inspur deployment runbook and resource caps. - Long-running worker supervision. - Disaster recovery, backup/restore checks, retention jobs, and evidence lifecycle controls. ## Phase 7: Self-Improvement With Guardrails - Coverage gap detector. - Source discovery recommendations. - Candidate source scoring. - Sandbox parse tests. - Human approval for high-risk source onboarding. - Evaluation dashboard: freshness, coverage, extraction quality, dedupe rate, source reliability. ## Non-Negotiables - Bun + TypeScript. - Modular adapters. - API-first contracts. - Provenance on every claim. - Darknet/leak collection is metadata-only until explicitly approved otherwise. - No auth bypass, CAPTCHA solving, private group joining, stolen data downloads, or threat actor interaction. - No unbounded queues, unbounded RAM caches, silent policy bypasses, or provenance-free intelligence.
