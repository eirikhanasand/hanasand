# Scraper Coordination

## Operating Rules
- Runtime: Bun + TypeScript.
- Keep work compact, tested, and production-directed. Avoid marketing copy and speculative feature bloat.
- Agents should own disjoint files where possible. If touching shared contracts, update this file first.
- All adapters must respect `src/policy/collectionPolicy.ts`.
- Darknet collection starts metadata-only. Do not implement credential bypass, CAPTCHA solving, threat actor interaction, or leaked-data downloads.
- Leak, victim, and threat-actor claim sources must not disappear as policy rejections when safe defensive metadata can be captured. Policy blocks for raw leaked data, downloads, credentials, auth bypass, CAPTCHA, private access, or threat-actor interaction remain hard blocks, but source-level approval/legal/status gaps should surface as metadata-only review or queue states with victim/company, claimed accounts, dataset size, actor statement/description, provenance, hashes, and allowed-field visibility.
- Every extracted claim must preserve provenance: source id, capture id, URL/channel, collection timestamp, content hash, extractor version, and confidence.
- Analyst-loop persistence is reserved for safe workflow metadata only: collection plans/runs/tasks, metadata review tasks, victim-notification packets, and claim ledger entries. These records may contain company/victim, affected-account counts or descriptions, dataset-size claims, actor statement summaries, hashes, provenance, and allowed actions, but must never contain raw leaked rows, credential values, private access material, downloaded datasets, or actor-interaction transcripts.

## Integration Contract
- Adapters return `AdapterRunResult`.
- New source types must be added to `SourceType`.
- Sources now use an expanded lifecycle in `SourceStatus`: `candidate`, `needs_review`, `approved`, `active`, `probation`, `degraded`, `quarantined`, `paused`, `disabled`, `retired`, and `rejected`.
- Medium, high, and restricted sources require explicit governance approval fields before they can become `active`; darknet metadata sources must also stay `metadataOnly`.
- Source records may include `health`, `scoring`, `governance`, `crawlState`, and `tags` sections. Consumers should treat missing sections as unset defaults, not as approval.
- Agent 01 owns `SourceReviewDecision` for source approve/reject/expire/quarantine/restore governance actions. Agent 09 source-admin APIs should accept this shape when adding review endpoints.
- Agent 01 seed bundles live under `seeds/` as versioned JSON fixtures for safe public CTI sources only. Importers validate and report duplicates by tenant/type/canonical URL; they must not trigger live crawling.

Agent 09 `/v1/sources` and future `/v1/intel/search` responses should preserve these fields for source coverage explanations.
